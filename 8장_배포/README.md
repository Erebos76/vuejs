## 배포: 서비스를 시작할 시간! 

- 개요
  - 애플리케이션을 서버에 배포하는 방법
    - 애플리케이션에 CI(Continuous Integration) 지속적 통합
      - 트래비스(travis)를 통한 CI구축
    - CD(Continuous Deployment) 지속적 배포 적용	
	  - 헤로쿠(Heroku)를 통한 CD구축

### 소프트웨어 배포
##### 배포 : 해당 프로그램을 사용 가능하도록 만드는 모든 행동

npm이나 ngrok를 통하여 외부에서 접근 가능
  - 개인 피시를 지속적으로 켜놓아야 함
  - 전원을 끌 경우 네트워크 유실로 애플리케이션에 접근이 불가능
  - 의미없는 url을 가짐

이를 해결하기 위해
- AWS를 통해 애플리케이션 복사
- 도메인을 구입하여 서버IP와 도메인을 연결하여 사용
  - 비용 발생

그렇기 때문에 헤로쿠, 트래비스를 사용하여 배포를 해보자

#### 배포를 위한 필요 절차
    어플리케이션을 위한 깃허브 저장소를 만들고 소스코드 업로드
	트래비스를 사용해 CI 설정
	애플리케이션을 헤로쿠와 연결
	헤로쿠가 애플리케이션을 실행하고 외부에 공개할 수 있도록 설정

## 깃허브
 - 규모가 작거나, 개인을 위한 공개/비공개 프로젝트를 위해 사용
 - 대형 기업들이 프로젝트 및 코드 리뷰, CI 등과 같은 개발 관련 활동에도 사용
 - 오픈소스의 성지

## 트래비스
[공식 홈페이지](https://travis-ci.org/) 
 - 깃허브를 위한 도구
 - 깃허브 프로젝트를 연동해 해당 프로젝트의 품질을 보장
 - 프로젝트 테스트를 수행하며, 빌드 성공유무를 알려줌
    - 트래비스로 관리되는 [Laravel](https://github.com/laravel/laravel) 
    
    
       현존하는 대부분의 개발언어를 지원
       
## 헤로쿠 
[공식 홈페이지](https://www.heroku.com/) 
 - 앱 배포하기 위한 클라우드 플랫폼
 - 애플리케이션을 만들고 깃허브 프로젝트와 연결하면 됨
    - 변경사항이 푸시될때마다 헤로쿠가 재배포를 실행함
